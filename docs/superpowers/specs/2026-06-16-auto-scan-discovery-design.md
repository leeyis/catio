# 自动扫描发现主机/数据库 — 设计文档

- 日期：2026-06-16
- 状态：已实现（2026-06-16 起按实测反馈迭代，下文已同步最终行为）
- 范围：在 Catio 中新增「自动扫描」功能，给定网段/地址范围并行发现主机与数据库节点，识别服务类型与版本、用凭证字典试登录，确认后一键入库或导出清单。

## 1. 背景与目标

当前添加主机(SSH)和数据库连接都是在「新建连接」弹窗里逐个手工录入。当面对一整片内网（如截图中的 `192.168.10.x` / `192.168.20.x`）时效率低。

目标：给定网段/地址范围 → 并行发现相关主机和数据库节点 → 识别服务类型+版本 → 用预先提供的凭证字典自动试登录 → 在结果页确认后一键入库（或导出清单）。

这是面向**用户自有基础设施**的运维发现工具（发现 + 自动试登录）。仅用于自有资产。

### 非目标 (YAGNI)
- 不做全端口(1-65535)扫描；只扫所选模式/引擎相关端口 + 用户自定义端口。
- 不做凭证的笛卡尔积组合（用户/密码分离列表）；字典是显式成对的 `用户名 密码`。
- 不做扫描结果的持久化历史/定时巡检；扫描结果仅存在于当前向导会话。
- 不新增独立的凭证存储机制：沿用现有"会话内存 + 启用账户验证时加密 vault"模型（命中凭证入库后按该模型保留，使首连免密）。

## 2. 关键决策（已评审确认）

| 维度 | 决策 |
|------|------|
| 识别深度 | 协议握手探测：识别服务类型 + 版本（主流协议免凭证即可识别） |
| 模式 | 主机 / 数据库 **二选一**，扫描前先选，不混扫 |
| 数据库引擎 | 扫描前从 40+ 引擎中勾选**一个或多个** |
| 凭证来源 | 扫描前提供凭证字典：手输 `用户名 密码`（每行一组）或上传字典文件 |
| 主机密钥 | host 模式额外支持私钥：上传一/多个私钥文件 + 用户名列表，试 `用户名×私钥` 组合（一个私钥可匹配多台主机） |
| 试登录 | 扫描中用字典逐个试登录，每节点**命中即停**；单次试登录有超时（12s），防止防火墙/tarpit 主机挂起 |
| 未认证节点（**主机/数据库分流**） | **主机**：只收录"能正常登录"的节点——识别到 SSH 但凭证/私钥均未命中、以及非 SSH 端口**一律不进结果**。**数据库**：识别到的库**仍列出**，未命中凭证者可入库为**草稿**并标记「需要认证」（首连补密码）；均支持导出 |
| 已存在节点 | 与 Vault 去重（与侧栏同源、按当前用户作用域），默认不勾 |
| 结果过滤 | 状态、引擎两个**下拉框**联合过滤 |
| 导出 | **导出选中**（仅勾选行），CSV/JSON |
| 容器形态 | **全屏接管主面板**向导，顶部 4 步步骤条 |
| 后端方案 | **方案 C 混合**：tokio 扫端口 + 主流原生探针 + JDBC 引擎驱动兜底 + 复用现有 `ssh_test`/`db_test_connection` 做认证 |
| 已认证入库 | 命中凭证载入**当次会话内存**，首连直连可用；启用账户验证（vault 已解锁）时，密码再**加密持久化到 vault**，重启/重新登录后首连仍免密。私钥命中存空口令标记，首连直接用私钥免口令登录 |

## 3. 用户流程

```
首页右上「自动扫描」按钮
        │
        ▼
┌─ 全屏向导（左侧 Vault 保留，顶部步骤条，可随时「返回」退出）─┐
│  ① 选模式      → 主机 / 数据库（二选一）；数据库勾选 1+ 引擎      │
│  ② 范围+凭证    → 扫描范围、端口、凭证字典（含密钥）             │
│  ③ 扫描中       → 实时进度，边扫边出结果，可取消                 │
│  ④ 结果确认     → 过滤/勾选 → 选分组 → 一键入库 / 导出选中        │
└──────────────────────────────────────────────────────────┘
```

## 4. 各步骤详细设计

### 步骤 ① 选模式
- 二选一：**主机** / **数据库**。
- 选「数据库」时展开引擎多选（复用 `src/services/dbEngines.ts` 的引擎编目与分组），勾选的引擎决定默认扫描端口集与探针策略。
- 选定后「下一步」进入 ②。

### 步骤 ② 范围 + 凭证

**扫描范围**（文本域，支持每行/逗号分隔混合）：
- CIDR：`192.168.10.0/24`
- 区间：`192.168.10.1-254`
- 单个 IP：`192.168.10.3`
- 主机名/域名：`db.internal.example.com`（DNS 解析为 IP）

**端口**：
- 默认 = 所选模式/引擎的默认端口（主机 22；数据库按每个所选引擎的默认端口，来自 `dbEngines.ts`）。
- 可追加自定义端口（如额外 SSH `2222`、MySQL `3307`）。

**凭证字典**（通用）：
- 手输文本域：每行一组 `用户名 密码`，按**第一个空白**切分（首段=用户名，其余整体=密码，兼容含空格密码）。
- 或上传字典文件（同样按行解析）。
- 字典**共享**：对每个识别出的节点，依次尝试所有凭证，命中即停。

**主机模式额外（密钥）**：
- 上传一个或多个私钥文件。
- 「密钥用户名」列表（逗号分隔，如 `root,ubuntu,admin`）。
- 对每台主机尝试 `用户名 × 私钥` 的所有组合。

**风险提示**：② 页顶部固定提示——批量试登录可能触发目标账号锁定策略，请仅对自有资产使用。

### 步骤 ③ 扫描中
- 展示实时进度：已扫描/总目标数、已发现数、用时。
- 结果**边扫边出**（流式追加行）。
- 「取消」按钮终止整批扫描（`scan_cancel`）。

### 步骤 ④ 结果确认
- 顶部汇总：`共发现 N · ✓已认证 a · ⚠未认证 b · 已存在 c`。
- 过滤条（两个下拉框联合过滤，仅影响显示与「全选」范围）：
  - 状态：全部 / ✓已认证 / ⚠未认证 / 已存在
  - 引擎：全部 / 各引擎（带计数）；主机模式下此下拉可隐藏或显示 OS。
- 表格列：`勾选 · 地址:端口 · 引擎(或 OS) · 版本 · 命中凭证(密码打码) · 状态`。
  - 主机模式：「引擎」列显示 OS/SSH banner；「命中凭证」显示 `user / ••••` 或 `user / 🔑密钥名`。
- 状态语义（行尾标签 + 悬浮说明）：
  - **✓ 已认证**：试出可用凭证，**默认勾选**；入库后命中凭证载入会话内存（启用账户验证时再持久化到 vault），首连直连可用。
  - **⚠ 未认证**：识别出服务但字典未命中。**仅数据库模式出现**（主机模式不收录未认证节点）；db 默认勾选，入库为**草稿**并标「需要认证」，首连补密码、成功后清除标记。
  - **端口开放·未确认**（open）：仅端口开放/JDBC 未确认，默认不勾，可手动勾选入库为草稿。
  - **已存在**：与 Vault 现有连接重复，灰显、默认不勾。
- 默认勾选：主机=已认证（主机结果全为已认证）；数据库=已认证 + 未认证。已存在/未确认默认不勾，可手动勾选。
- 操作区：
  - 分组选择（复用 `src/state/groups.ts`），入库时写入 `profile.group`。
  - 「一键入库（已选 N）」：复用 `saveProfile`（host）/ `saveDbConnection`（db）写入 Vault。
  - 「导出选中」：仅导出勾选行，CSV / JSON。作为凭证发现清单，**导出明文用户名/密码**（命中的工作凭证，供自有资产留存）；屏幕表格仍对密码打码，仅导出文件携带明文。

## 5. 数据流与状态

- 扫描结果仅存于向导组件状态，退出即弃（不持久化）。
- 入库：
  - 主机 → `src/state/connections.ts` 的 `saveProfile`（host/port/user/auth/group）。
  - 数据库 → `src/state/dbConnections.ts` 的 `saveDbConnection`（dbType/engineId/host/port/user/database?/group/needsAuth?）。
  - ✓已认证：命中 secret 写入会话内存（`src/state/sessionSecrets.ts`，键到新建 profile id）；启用账户验证时再经 `rememberConnSecret` 加密持久化到 vault。私钥命中写空口令标记。
  - ⚠未认证 / 未确认（仅数据库草稿）：只写 profile + `needsAuth=true`，不写 secret；侧栏标「需要认证」，首连补密码、成功后清除标记。
- 去重键：`地址:端口`（数据库再带 `engineId`/`dbType`）与 Vault 现有连接比对；去重基线**与侧栏同源**（按当前用户作用域，避免"侧栏看不到却判已存在"）。

## 6. 后端架构（方案 C 混合）

新增模块 `src-tauri/src/scan/`：

- `range.rs`：将扫描范围文本展开为目标列表（CIDR / 区间 / 单 IP / 域名解析），与端口集做笛卡尔积得到 `(ip, port)` 待扫列表。
- `probe.rs`：原生协议探针——
  - 主流协议族免凭证识别类型+版本：SSH（读 banner）、MySQL-wire（读握手包，覆盖 MySQL/MariaDB/TiDB/Doris 等）、PostgreSQL-wire（覆盖 PG/openGauss/CockroachDB 等）、Redis（`PING`）、MongoDB（`isMaster`）。
  - JDBC-only 引擎（达梦/Oracle/DB2/…）无原生 Rust 握手：跳过原生探针，靠「用字典凭证试连一次」确认；连不上则标「端口开放·未确认」。
- `commands.rs`：
  - `scan_start(args) -> scanId`：开 tokio 并发任务池（信号量限流，默认 64）执行扫描，结果通过事件流推送。单次试登录 12s 超时。**主机模式只对试登录成功的节点 emit `scan://found`**（未认证/非 SSH 不上报）；数据库模式对识别到的库都上报（authed/unauthed/open）。
  - `scan_cancel(scan_id)`：取消整批（`CancellationToken`）。
  - `scan_read_text_file(path) -> String`：读取本地文本字典文件（②页"上传字典文件"用，沿用"前端选路径→Rust 读取"模式，避免前端引入 plugin-fs）。

**认证**：统一复用现有 `ssh::conn::ssh_test` 与 `db::commands::db_test_connection` 逐个试字典凭证、命中即停——不重造认证逻辑，吃现有成熟驱动栈（含 JDBC/达梦）。

**并发与安全**：
- 并发上限默认（如同时 64 个目标在飞），可在 ② 页或配置中调整。
- 每节点命中即停，限制单节点试登录次数。
- 仅对识别出匹配服务的端口执行试登录。

**事件流（前端监听）**：
- `scan://progress` — `{ scanId, scanned, total, found }`
- `scan://found` — 单条发现结果 `{ ip, port, engine|os, version, status, hitUser?, hitAuthKind? }`
- `scan://done` — `{ scanId }`

## 9. 边界情况

- 密码含空格：按第一个空白切分，其余整体为密码。
- 字典空：无凭证仍可扫描+识别。**主机**：因无凭证命中，结果列表为空（主机只收录可登录节点）。**数据库**：识别到的库全部列为 ⚠未认证草稿（标「需要认证」）。
- 同一 IP 同时开多个相关端口：每个 `(ip, port)` 各算一行结果。
- 域名解析失败：跳过并在进度中计为失败/跳过。
- 已被 Vault 收录：标「已存在」，默认不勾，不重复入库。
- 取消扫描：已发现结果保留，可继续在 ④ 操作。

## 10. 验收标准

1. 选主机模式、输入一个 `/24` 网段 + `root pass` 字典，能并行扫出开放 22 端口的主机；**结果列表只含命中凭证(✓已认证)、能正常登录的节点**（未命中/非 SSH 不出现），并识别出 OS/SSH banner。
2. 选数据库模式、勾选 MySQL+PostgreSQL，能识别出对应服务与版本；字典命中标 ✓，未命中标 ⚠（仍列出、可入库为「需要认证」草稿）。
3. 达梦(JDBC)引擎：用正确凭证能识别并标 ✓；无凭证标「端口开放·未确认」，可入库为「需要认证」草稿。
4. 结果页状态/引擎下拉过滤、勾选、导出选中(CSV/JSON) 均可用；导出为凭证清单，含明文用户名/密码（屏幕仍打码）。
5. 一键入库后，✓已认证连接首连直连可用（会话内 + 启用账户验证时跨重启）；⚠/未确认草稿在侧栏标「需要认证」，首连弹凭证、成功登录后清除标记。
6. 已在 Vault 的连接被识别为「已存在」且默认不勾、不重复入库（去重与侧栏同源）。
7. 扫描可取消；UI 支持中英文与主题切换。
